## Why I Finally Did It

I realized how much a researcher needs a single place online: publications, project pages, fieldwork photos, and occasional thoughts—in both English and Chinese.

I had put it off for six months because every option felt like a compromise: WordPress meant plugin maintenance, Notion locked content into a proprietary format, and coding from scratch seemed like a waste of energy.

## The Tech Stack

My first attempt used **Material for MkDocs**. The upsides were clear: pure Markdown content, a built-in i18n plugin, and a live dev server. I built the skeleton in a weekend—homepage, publications, about, five research project pages, and a Leaflet.js map.

But problems surfaced quickly:

- MkDocs' blog plugin and i18n plugin were incompatible; the listing page rendered blank
- Custom CSS was buried under theme layers; changing one color required overriding three files
- The mobile navbar had poor contrast in dark mode

A few days later, I migrated to **Hugo + Hextra**. The reasons were practical: Hugo's template system is more flexible, Hextra's design feels more modern, and the build speed is an order of magnitude faster than MkDocs.

The migration was surprisingly smooth—content files moved from `docs/` to `content/`, config converted from `mkdocs.yml` to `hugo.yaml`, and most Markdown files needed no changes. The only real work was re-tuning the CSS color palette.

## The Workflow

The entire development happened inside [Claude Code](https://claude.ai/code), with all communication in Chinese. My pattern:

1. Describe what I want in Chinese (e.g., "compress the homepage into a business card")
2. It proposes two or three options with trade-offs
3. I pick one, or keep refining
4. It writes code while I watch
5. Local preview; iterate if it doesn't feel right

This rhythm works for me because I don't need to context-switch languages. Technical terms in English are handled naturally, but subjective judgments like "this button is too far right" are more precise in Chinese.

## Why Posts, Not Blog

The word "blog" carries expectations—regular updates, consistent themes, substantial length. I don't want to commit to those rules. Sometimes I just want to drop a blurry fieldwork photo with a caption like "this tree is aggressively confident." Sometimes I read half a book and want to jot two fragmented notes. These aren't blog posts; they're just posts.

This section will eventually hold reading notes, fieldwork scribbles, urban observation photos, tool rants, and half-baked ideas. No commitment, no expectation, no KPI.
